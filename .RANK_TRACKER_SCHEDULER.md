# Rank Tracker Scheduled Checks Implementation

## Overview
Added automated weekly rank checking for all users with active Rank Tracker subscriptions.

## What Was Built

### `scheduleRankChecks` Cloud Function
**Location:** `functions/src/index.js` (line 1510+)

**Trigger:** Every Monday at 2 AM UTC  
**Type:** Firebase Scheduled Function (onSchedule)

### How It Works

1. **Find eligible users**: Queries all users with `subscriptions.rankTracker.active = true`

2. **Get their keywords**: For each user, retrieves all tracked keywords from `trackedKeywords` collection

3. **Smart rate limiting**:
   - Only checks keywords not updated in the past 7 days (avoids duplicate queries)
   - Adds 1-second delay between SerpAPI calls
   - Respects 5K/month SerpAPI quota (~166 checks/day spread across week)

4. **Perform rank checks**: For each keyword:
   - Calls SerpAPI with keyword, location (city, state), and device type
   - Searches organic results for the target domain
   - Checks for presence in Google Maps local pack (3-pack)
   - Updates `trackedKeywords` doc with new rank + `lastChecked` timestamp
   - Appends entry to `rankChecks` collection for history/analytics

5. **Error handling**:
   - Catches individual keyword failures without stopping other checks
   - Logs success/failures to Cloud Functions logs
   - Returns summary: { totalChecks, successCount, failureCount, timestamp }

## Key Features

✅ **Automatic**: Runs every Monday 2 AM UTC, no manual intervention  
✅ **Rate-limited**: Respects SerpAPI quota with intelligent scheduling  
✅ **Selective**: Only checks keywords that need updating (7-day window)  
✅ **Comprehensive**: Checks both organic ranks + local pack presence  
✅ **Observable**: Logs every check result for debugging/auditing  
✅ **Resilient**: Continues checking other keywords if one fails  

## Required Environment Variable

Must already be set in Firebase Console → Functions → Runtime environment variables:

```
SERPAPI_KEY = your_serpapi_api_key
```

If missing, function logs warning and exits gracefully (doesn't crash).

## Frontend Integration

✅ The existing `checkKeywordRank` onCall function remains unchanged  
✅ Users can still manually trigger checks from the UI (Keywords page)  
✅ Both manual + scheduled checks update the same data (rank history, lastChecked timestamp)

## Monitoring & Debugging

**Check logs via Firebase Console:**
```
Functions → Logs → filter by "scheduleRankChecks"
```

**View scheduled run history:**
- Firebase Console → Scheduler → Find "scheduleRankChecks"
- Shows last run time, next run time, execution results

**Example log output:**
```
Found 5 users with active rank tracker subscriptions.
User abc123: checking 12 keywords
✓ plumber near me: rank 5 (in local pack)
✓ emergency plumbing: rank 12
Skipping hvac services (checked 2 days ago)
...
Scheduled rank check complete: 8/10 succeeded, 2 failed
```

## Cost Impact

- **SerpAPI**: ~10 checks/day × 5 days = ~50 checks/week = ~200 checks/month
- **Cost**: $50/month plan covers 5,000/month, so well within quota
- Can scale to 5,000+ keywords if needed (spread over multiple runs)
